# 全局约束

## 语言偏好

- 所有回复必须使用简体中文
- 思考过程使用中文
- 代码注释使用中文（除非用户要求英文）
- 技术术语可保留英文，但首次出现时应提供中文解释
- 保持简洁、专业，避免冗余的翻译腔

## Git 提交规范

- **禁止自动提交**：除非用户明确要求，否则不要执行 git commit
- **提交格式**：遵循 Angular 规范，格式为 `<type>(<scope>): <subject>`
  - type: feat | fix | docs | style | refactor | perf | test | build | ci | chore | revert
  - 示例: `feat(auth): 添加用户登录功能`
- **禁止联合作者**：commit message 中不要添加 Co-authored-by 信息
- 提交前确保变更符合预期，必要时先展示 diff 确认

## Git 日志提取与总结约束

### 执行策略

- **日志溯源**：在回复日志摘要前，必须执行 `git log` 读取根目录的提交记录。
- **时间跨度**：**默认仅提取并总结今天的提交记录**。除非用户明确指定日期（如“昨天”、“上周”或特定日期），否则严禁跨日期读取。

### 内容聚合逻辑

- **逻辑归并**：**严禁流水账式陈述**。必须根据提交记录中的 `scope` 或功能关联性，将多次微小提交合并为一条完整的技术摘要。
- **技术导向**：使用极简、专业的技术语言。侧重描述“解决了什么技术难题”或“实现了什么功能模块”，而非简单的动作罗列。
- **噪音过滤**：自动过滤 `chore`、`style`、`docs` 等不影响核心逻辑的提交，除非其构成当日工作的主要内容。

### 输出格式规范

- **列表格式**：严格使用数字列表格式，严禁使用无序列表：
  1. xxxxx
  2. xxxxx
  3. xxxxx
- **语言风格**：保持极简、专业，去除所有冗余的修饰词、客套话或翻译腔。
- **术语处理**：保留原始技术术语（如 **SSE**, **WebSocket**, **Vulkan**, **GC**, **Diffing**），无需额外解释。

### 示例参考

1. **控制算法 (Control)**：重构了运动补偿逻辑，通过引入 **PID** 算法显著降低了执行机构的路径偏移。
2. **数据持久化 (Store)**：整合了关于 **SQLite** 的多项 Schema 变更，完成了任务队列的原子化存储方案。
3. **通信层 (Network)**：优化了 **SSE** 断线重连机制，解决了集群环境下高并发指令丢失的问题。
